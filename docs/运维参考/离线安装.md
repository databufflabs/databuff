# 离线安装

在无外网或内网隔离环境中，使用一体化离线包完成 Docker 全栈部署。

## 获取离线包

从 [GitHub Releases](https://github.com/databufflabs/databuff/releases) 或内网镜像站下载对应架构包：

- `databuff-ai-apm-offline-<version>-amd64.tar.gz`
- `databuff-ai-apm-offline-<version>-arm64.tar.gz`

包内包含：部署脚本、`ai-apm-stack` 应用镜像、`doris-stack` 基础设施镜像。发布前请核对附带的 checksum 文件。

## 安装步骤

```bash
tar -xzf databuff-ai-apm-offline-0.1.1-amd64.tar.gz
cd databuff-ai-apm-offline-0.1.1-amd64
sudo ./install.sh
```

`install.sh` 会：

1. 将部署文件解压到 `APM_INSTALL_DIR`（默认 `/opt/databuff-ai-apm`）
2. `docker load` 离线镜像（本地已有且版本一致时可跳过，除非设置 `FORCE_LOAD_IMAGES=1`）
3. 执行 `start.sh` 启动全栈（可用 `SKIP_START=1` 仅安装不启动）

安装完成后终端输出 Web UI、默认账号与 OTLP 地址。

## 环境变量

| 变量 | 说明 |
|------|------|
| `APM_INSTALL_DIR` | 安装目录，默认 `/opt/databuff-ai-apm` |
| `SKIP_START` | `1` = 仅安装不启动 |
| `FORCE_LOAD_IMAGES` | `1` = 强制重新 `docker load` |

## 与在线安装的区别

在线安装通过 `ai-apm-install.sh` 从 CDN 拉取部署包与分架构镜像包；离线安装**全程不依赖 curl/CDN/registry**，适合 U 盘搬运或 air-gapped 机房。

在线环境仍可使用一键命令：

```bash
curl -fsSL https://databuff.ai/databuff/ai-apm-install.sh | bash
```

## 故障恢复

| 问题 | 建议 |
|------|------|
| `docker load` 失败 | 检查磁盘空间；使用 `FORCE_LOAD_IMAGES=1` 重试 |
| 启动超时 | 见 [Docker 运维参考](Docker运维.md) 日志与排障 |
| 需换版本 | 停止服务后重新解压目标版本离线包执行 `install.sh` |
